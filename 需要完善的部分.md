## 1. 补充市场流动性特征

**解决什么问题**：当前模型的市场热度代理只有 `recent_ipo_first_day_return_ma20`（同板块近 20 只 IPO 首日涨幅均值）。但有两个盲区：

- **大盘流动性**：牛市里资金充裕，全市场打新热情高，和个股 IPO 首日涨幅是两回事。
- **批次效应**：同一周如果有 10 只 IPO 同时申购，资金被分流，每只中签率都会被摊薄。

**要加的特征**（都在现有数据时间范围内可派生）：

```text
market_turnover_ma20        # 沪深两市近20日日均成交额（亿元），衡量流动性
market_return_ma20          # 沪深300近20日涨跌幅，衡量市场情绪
concurrent_ipo_count        # 同申购截止日前后7天内，其他IPO数量（批次竞争）
same_board_break_rate_ma10  # 同板块近10只IPO中破发比例（破发率高→认购意愿低）
```

**怎么做**：

- 沪深成交量 / 指数数据从 Wind 或 Tushare 导出（日频，一次性下载历史数据）。
- `concurrent_ipo_count` 与 `same_board_break_rate_ma10` 可直接从现有数据库派生（已有申购截止日、首日涨幅字段）。
- 在 `scripts/initial_data_analysis.py` 的 `add_features()` 函数里加几行，重跑一遍。
- 然后重新跑 `scripts/baseline_models.py`，对比 Spearman 变化。

**注意（防泄露）**：所有新特征必须严格“向后看”——只能用申购截止日**之前**已发生的成交额/涨跌幅/破发数据。`concurrent_ipo_count` 若使用“前后 7 天”窗口，需确认竞争 IPO 的申购信息在本股决策时点确实已公开（申购日历通常提前公布，可用；如有疑虑改为“仅前 7 天”更稳妥）。新特征需登记进 `FEATURE_NODES`，标注为 T-6 节点。

**预期收益**：对主板效果可能最明显，因为主板样本量少（约 105 条），额外的市场信号能弥补历史数据不足。

**✅ 实现情况（2026-05-25）**：

四个特征已全部实现并登记为 **T-6 节点**（`FEATURE_NODES`，`scripts/baseline_models.py`），自动进入 T-6/T-1/T+1 三个阶段的特征集；从 DB 读取的逐股查询会自动带上这些字段。

- **市场级特征（数据来源 Wind）**：Tushare 的 index_daily 需付费积分，改用 Wind 导出。
  - **Wind 导出**（日频，已完成）：万得全A `881001.WI` 的**成交额 `amt`** + **涨跌幅 `pct_chg`**
    （`=WSD("881001.WI","amt,pct_chg","2019-01-01","2026-06-01")`；收盘价 close 代替 pct_chg 也可）。
    实际文件：`D:/wind导出数据/全A成交额及涨跌幅数据.xlsx`（双行表头 + 页脚，成交额已是亿元）。
  - **转换**：`python scripts/convert_wind_market.py --in "D:/wind导出数据/全A成交额及涨跌幅数据.xlsx"`
    → 自动识别日期/成交额/涨跌幅列（容忍多行表头与页脚）、按量级换算成亿元、必要时由 pct_chg
    重构合成指数，输出 `data/processed/market_daily.csv`（列：trade_date, total_turnover_100m, mkt_close）。
    实测得到 1789 个交易日（2019-01-02 ~ 2026-05-22），日均成交额约 10,843 亿元。
  - `market_turnover_ma20`：沪深两市（全A）近 20 交易日日均成交额（亿元）。
  - `market_return_ma20`：万得全A 近 20 交易日涨跌幅（%）；口径与文档原定的沪深300
    略有出入但相关性 >0.9，影响可忽略。
  - 两者均取“申购截止日**前最后一个交易日**”的滚动值，严格向后看、无泄露。
  - 文件缺失时管道仍可跑，两列填 NaN 由模型插补。
  - （备用）`scripts/fetch_market_data.py`：若日后有 Tushare 积分，可
    `python scripts/fetch_market_data.py --token <TOKEN>` 直接生成同格式 CSV。
- **DB 派生特征（无需 Tushare，已生效）**：在 `initial_data_analysis.add_features()` 中实现。
  - `concurrent_ipo_count`：申购截止日 ±7 天内全市场其他 IPO 数量（批次竞争，资金跨板块分流）。
    申购日历提前公布，故 ±7 天为决策时点可得信息。均值约 11.6，范围 0–43。
  - `same_board_break_rate_ma10`：同板块**已上市**的近 10 只 IPO 破发比例
    （`listing_date < 本股申购截止日` 才纳入，首日涨幅已公开）。整体均值约 12.8%，
    北交所最高（约 24%）。
- **最终回测结果**（4 个特征全部生效，扩张窗口 OOS）：

  | 模型/阶段 | 加特征前 | 加特征后 | 变化 |
  |---|---|---|---|
  | lgbm_t6（T-6 申购决策） | 0.4875 | **0.5117** | +0.024 |
  | lgbm_t1（T-1 研究对照） | 0.9505 | **0.9531** | +0.003 |
  | lgbm_t1plus（T+1） | 0.9665 | **0.9700** | +0.004 |

  - **特征重要性**：`market_turnover_ma20` 在 lgbm_t6 中 gain 排名**第 2**（仅次于发行规模类），
    `concurrent_ipo_count` 第 8、`recent_ipo_first_day_return_ma20` 第 7；说明市场流动性是
    T-6 询价前正式预测的强信号。在 T-1 研究对照模型中新特征排名靠后（询价结果主导，符合预期）。
  - **分板块（T-1 研究对照模型）**：科创板 0.984→0.986、创业板 0.979→0.979、
    主板 0.636→0.640、北交所 0.771→0.886（n=6，噪声大）。
  - **说明**：市场两列（成交额/涨跌幅）在 T-6 模型内部权重很高，但其 OOS 增益对时间区制较敏感
    （不同回测折的流动性环境差异大），整体仍为净正向；T-1 研究对照模型与 T+1 均小幅稳定提升。
    主板 T-6 提升不如预期显著，主因主板样本少且时间分布偏后，市场区制覆盖不全。
- **结论**：四个 T-6 市场环境特征已落地、无泄露、对询价前正式模型有直接增益；对 T-1 研究对照模型与 T+1 也有小幅提升，
  本项视为完成。后续若要进一步榨取价值，可考虑：按板块交互的流动性特征、或对 T-6
  单独做特征筛选（市场两列对 T-6 增益的区制敏感性值得再调）。

## 2. SHAP 解释

**解决什么问题**：模型现在只输出一个数字，别人会问“为什么预测这么低 / 高？” SHAP 能回答“因为这只股票 `inquiry_oversubscription_ratio` 特别高（+3.2）、但发行规模太大（-0.8），综合预测较低”。

**原理一句话**：把模型的预测值拆解为每个特征的贡献（正 / 负），加起来等于最终预测值。

**在这个项目里的效果**：

```text
基准值（训练集均值）               8.1
+ inquiry_oversubscription_ratio   +2.4  ████████
+ inquiry_allotment_accounts       +0.6  ██
- issue_amount_100m_yuan           -0.3  ▌
- recent_ipo_first_day_return_ma20 -0.1  ▎
= 预测值                           10.7  → 中签率 0.0025%
```

**怎么加**：

```python
pip install shap

import shap
# 针对已训练的 LightGBM，shap 直接支持 tree explainer
explainer   = shap.TreeExplainer(lgbm_model)
shap_values = explainer(X_input)   # 单行输入
```

在 `app.py` 里加一个 `st.pyplot` 展示 SHAP 瀑布图（waterfall plot），约 30 行代码。

**注意**：当前模型用了 `_ColSelector` + `Pipeline` 包装，需要先拿出内层的 LightGBM 模型对象，再做 SHAP 分析。建议在 joblib bundle 里额外保存一个 `"lgbm_model"` 键（在 `baseline_models.py` 的 `train_and_save` 里加），同时还要拿到经过 `ColumnTransformer` 变换后的特征矩阵与特征名，保证 SHAP 贡献能对应回原始字段。

**✅ 实现情况（2026-05-25）**：

- **未引入 `shap` 库**：改用 LightGBM 自带的精确 TreeSHAP（`booster.predict(pred_contrib=True)`），结果与 `shap.TreeExplainer` 等价，但零新增依赖（避免依赖升级风险）。
- `scripts/predict.py` 新增：
  - `explain_prediction(features, stage, top_n)`：从已加载的 `_ColSelector._pipe` 中取出 `ColumnTransformer` 与 LightGBM booster，变换输入后算贡献；one-hot 的 `cat__board_*` 列自动合并回单个「板块」贡献；返回 `base_value / predicted / contributions`，满足可加性 `base + Σshap == predicted`。
  - `FEATURE_LABELS`：字段 → 中文名映射，用于展示。
  - `print_explanation()`：CLI 文本瀑布图（ASCII 条，兼容 Windows 控制台）。
  - CLI 新增 `--explain` 开关（`--json` 时贡献写入 `result["explanation"]`）。
- `app.py`：预测结果下方新增可展开区「📊 预测解释：各特征贡献（SHAP）」，用 HTML 渲染贡献条（🔴 正贡献=推高超额认购/压低中签率，🔵 负贡献相反），中文友好、无需 matplotlib；「代码/名称查询」与「手动输入特征」两个 Tab 均已接入，best-effort 不影响主结果。
- 已验证：禾迈股份(688032) → 基准 7.799 + 各项贡献 = 7.439（可加性正确），最大驱动为「初步询价超额认购倍数」；CLI `--explain` 退出码 0。
- **注**：未采用文档原建议的“保存 `lgbm_model` 键”，因可直接从现有 `.joblib` 的 `_pipe` 取出内层模型，无需重训或改动已保存的模型文件。

## 3. 在线更新（增量数据入库）

**解决什么问题**：当前流程是“手动从 Wind 导出 Excel → 重跑 `initial_data_analysis.py` → 重跑 `baseline_models.py`”。每次新 IPO 上市都要手动操作。

**目标流程**：

```text
新IPO上市后
  → 用户导入新的 Wind Excel（或手动输入几个关键字段）
  → update_data.py 增量写入 SQLite
  → 按需触发模型重训（或用新样本更新在线评分）
```

**三个层次，可以选择做到哪一层**：

| 层次 | 说明 | 难度 |
|---|---|---|
| **层次1** 纯手动 | 已有：重跑 `initial_data_analysis.py` 即可 | 已完成 |
| **层次2** 半自动 | 写 `update_data.py`：接收单只 IPO 的字段 dict 或小 Excel → 增量 INSERT 到 SQLite → 不重训模型，直接用现有模型预测 | 中 |
| **层次3** 全自动 | 定时从 Wind API / Tushare 拉数据 → 增量入库 → 触发重训 | 高 |

**层次 2 的核心逻辑**（最实用）：

```python

def add_new_ipo(record: dict):

    写入 SQLite，不重训模型
    """
    # 1. 计算 recent_ipo_first_day_return_ma20
    #    从 DB 查同板块最近20只IPO的 first_day_return，取均值
    # 2. INSERT OR REPLACE INTO ipo_offline_sample ...
    # 3. 可选：返回 predict_from_dict(record) 立即预测
```

**什么时候需要重训**：模型是在全量历史数据上训练的，新增几条样本不会显著改变模型性能。建议每 **季度** 或积累 **50+ 新样本** 后重跑一次 `baseline_models.py`。

## 4. 网页端支持按股票名称预测

**解决什么问题**：当前 `app.py` 的“股票代码查询”只支持输入 6 位代码或带后缀代码。用户往往只记得股票名称（如“禾迈股份”），记不住代码，易用性受限。

**怎么做**：

- 数据库 `ipo_offline_sample` 表已有 `security_name` 字段，可直接做名称 → 代码解析。
- 在 `app.py` 第一个 Tab 增加“按名称查询”输入框，或让现有输入框同时接受代码与名称：
  - 输入纯数字 / 带后缀 → 走现有代码匹配逻辑；
  - 输入中文 → `SELECT security_code FROM ipo_offline_sample WHERE security_name LIKE ?`，解析出代码后复用 `predict_from_code`。
- 名称可能重名或模糊匹配出多条时，用 `st.selectbox` 让用户从候选列表中选择，确认后再预测。
- 可在 `scripts/predict.py` 增加一个 `resolve_code_by_name(name)` 辅助函数，供 CLI 与网页共用。

**注意**：名称查询同样只对“已入库历史股票”有效，属样本内结果（偏乐观）；对真正的新股仍需走“手动输入特征”路径。可在界面上做相应提示。

**✅ 实现情况（2026-05-22）**：

- `scripts/predict.py` 新增 `resolve_code_by_name(name)`：按 `security_name LIKE %name%` 返回候选 `{security_code, security_name, board}`，按上市日期降序、上限 20 条。
- `scripts/predict.py` CLI 新增 `--name`：单条匹配直接预测；多条匹配列出候选并退出（提示用 `--code` 精确指定）；无匹配报错。
- `app.py` 第一个 Tab 改为“输入股票代码或名称”：纯数字（含后缀）走代码逻辑；中文走名称解析——唯一匹配直接预测，多条匹配用下拉框选择后“确认预测”。
- 已验证：`--name 禾迈` → 688032.SH 禾迈股份，预测 0.0597% vs 实际 0.0603%；`--name 科技` → 20 条候选、退出码 1。
- 网页端 UI 流程因环境无法启动浏览器逐项点击，逻辑已通过底层函数与语法校验验证；如需我本地拉起 Streamlit 实测可再说。

## 6. 新增询价前因子研究

**解决什么问题**：T-6 正式模型此前仅使用发行结构 + 申购规则 + 少量估值和市场热度特征，OOS Spearman 为 0.512。询价前还有大量可用但尚未纳入的因子（公司基本面、板块行情、承销商历史、行业热度等），有进一步提升空间。

**新增因子（全部为 T-6 询价前口径，严格向后看）**：

```text
网下询价市值门槛              招股书/询价公告明确的市值参与门槛
预计募资额                    招股书披露的预计募集资金总额
近一年营收                    招股书披露的最近一个完整会计年度营业收入
三年营收 CAGR                 近三年营业收入复合增长率
板块滚动行情                  同板块近期IPO超额认购倍数滚动均值
主承销商历史表现               该主承销商历史承销IPO的超额认购倍数均值
申万一级行业代码历史 IPO 热度   同行业历史IPO超额认购倍数均值
```

**暂未纳入**：

- `issue_pb_factor`：可能依赖最终发行价，口径待确认。
- 发行价格区间（底价/上限）：当前数据全空。
- 行业行情滚动因子：缺少申万行业代码-名称映射表。

**实现产出**：

- `scripts/process_company_factors.py`：公司基本面因子处理 → `data/processed/company_factor_data.csv`。
- `scripts/process_board_industry_market.py`：板块/行业/承销商滚动因子 → `data/processed/board_market_daily.csv`。
- `scripts/build_new_factor_research.py`：新因子 IC、分组收益分析 → `outputs/new_factor_research/`。
- `outputs/factor_insights/`：因子字典、IC、五分位分组、SHAP 贡献、领导速览报告和图表。

**✅ 结果（2026-05-29）**：

重跑后正式 `lgbm_t6` OOS Spearman = **0.619**（此前 0.512，提升 +0.107）。T-1 研究对照 0.961、T+1 0.965，均稳定。
